## 2. CSV 文件格式规范

### 2.1 习题文件 (`data/exercises/EX-{id}.csv`)

```csv
# exercise_id: EX-20260624-001
# type: mixed
# created_at: 2026-06-24T10:30:00
# total: 50
index,num1,operator,num2,correct_answer
1,15,+,7,22
2,22,+,33,55
3,88,-,21,67
```

### 2.2 答案文件 (`data/answers/EX-{id}_answers.csv`)

```csv
# exercise_id: EX-20260624-001
# student: XiaoMing
# submitted_at: 2026-06-24T20:00:00
problem_index,student_answer
1,22
2,55
3,67
```

### 2.3 成绩文件 (`data/scores/scores.csv`)

```csv
exercise_id,graded_at,total,correct,wrong,percentage,wrong_indices
EX-20260624-001,2026-06-24T21:00:00,50,45,5,90.0,"1,5,12,23,41"
EX-20260625-001,2026-06-25T20:30:00,50,48,2,96.0,"15,33"
```

### 2.4 弱项分析输出 (`data/analysis/weak_problems.csv`)

```csv
num1,operator,num2,correct_answer,wrong_count,total_attempts,error_rate
15,+,7,22,3,5,60.0
22,+,33,55,2,4,50.0
88,-,21,67,2,3,66.7
```

## 3. 数据模型

| 模型 | 属性 | 说明 |
|------|------|------|
| `Exercise` | id, type, problems[], created_at | 一次习题集 |
| `AnswerSheet` | exercise_id, student, answers{} | 一份答卷 |
| `Score` | exercise_id, total, correct, wrong, percentage, wrong_indices | 一次成绩 |
| `StudentRecord` | name, scores[], history | 学生成绩档案 |

## 4. 关键技术应用

| 技术 | 应用位置 | 说明 |
|------|----------|------|
| **CSV 存储** | `csv_handler.py` | 习题/答案/成绩的持久化 |
| **防御性编程** | `csv_handler.py` | 文件不存在、格式错误、数据类型校验 |
| **正则表达式** | `csv_handler.py` | 解析元数据行 `# key: value`、校验答案格式 |
| **表驱动编程** | `exercise_builder.py` | 习题类型→运算符/约束映射表 |
| **契约式编程** | `grader.py` | 前置条件(答案完整性)、后置条件(分数一致性)、不变量 |
| **数据结构** | `models.py` | OrderedDict 答案映射、Counter 弱项统计 |
```

```python
# Grader.grade() 的契约：
#   前置条件: exercise 和 answer_sheet 非空，problem_count 一致
#   后置条件: correct + wrong == total，0 <= percentage <= 100
#   不变量:   wrong_indices 中每个索引都对应一道确实答错的题
```

## 7. 防御性编程策略

| 异常场景 | 处理方式 |
|----------|----------|
| CSV 文件不存在 | 明确错误消息 + 建议路径 |
| CSV 格式损坏 | 行号定位 + 显示问题行 |
| 答案数量不匹配 | 列出缺失/多余的索引 |
| 答案非数字 | 正则预检 + 问题定位 |
| 分数不一致 | 自动重算 + 告警 |
